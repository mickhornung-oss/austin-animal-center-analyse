# Contributing Guidelines

Thank you for your interest in contributing to austin-animal-center-analyse! This document provides guidelines and instructions for contributing to this project.

## Getting Started

### Prerequisites
- Python 3.8+
- Jupyter Notebook
- pandas, scikit-learn, matplotlib, seaborn

### Local Setup
1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/austin-animal-center-analyse.git`
3. Navigate to the project: `cd austin-animal-center-analyse`
4. Create a virtual environment: `python -m venv .venv`
5. Activate it: `.\.venv\Scripts\Activate.ps1` (Windows) or `source .venv/bin/activate` (Linux/Mac)
6. Install dependencies: `pip install -r requirements.txt`

## Development Workflow

### Creating a Feature Branch
```bash
git checkout -b feature/your-feature-name
```

### Code Guidelines
- Follow [PEP 8](https://www.python.org/dev/peps/pep-0008/) for Python code
- Use meaningful variable and function names
- Add docstrings to functions and classes
- Keep notebooks well-organized with clear markdown sections

### Analysis Best Practices
- Document data assumptions
- Include visualizations for findings
- Provide statistical validation
- Comment on business implications

## Submitting Changes

### Commit Messages
- Use clear, descriptive commit messages
- Format: `type: brief description`
- Examples: `feat: add survival analysis`, `fix: data cleaning issue`, `docs: update findings`

### Pull Request Process
1. Push your changes to your fork
2. Open a Pull Request with:
   - Clear title describing the analysis or improvement
   - Explanation of findings or changes
   - Supporting visualizations or statistics
   - Reference to any related issues

3. Wait for review and address feedback
